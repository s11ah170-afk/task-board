# task-board プロジェクト ルール

## 基本ルール

- ファイルの削除・上書き前には必ず確認を求めること
- フォルダ名・ファイル名の日付は「YYYYMMDD」形式にすること
- 作業が完了したら「Claude_Work/作業ログ/」フォルダに実施内容を追記すること

## Git 運用ルール

- コードを変更するたびに、必ず GitHub へプッシュすること
- コミットは変更の単位ごとにこまめに行い、意味のあるコミットメッセージを日本語で書くこと
- コミットメッセージの形式: `種別: 変更内容の概要`
  - 例: `feat: タスク追加機能を実装`
  - 例: `fix: 完了チェックが反映されないバグを修正`
  - 例: `style: カードコンポーネントのレイアウトを調整`
- プッシュ先は `origin main`（または `origin master`）を基本とする
- 破壊的な操作（`--force` 等）はユーザーに確認を取ってから実行すること

## プロジェクト概要

- プロジェクト名: task-board
- 用途: タスク管理ボードアプリ
